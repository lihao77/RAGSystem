# 系统架构评估报告

> 评估日期: 2026-03-16
> 范围: 底层数据传输、工具系统、Agent 体系、SSE 通信

## 一、当前架构总体评价

整体架构设计合理，事件驱动 + ReAct 循环 + SSE 流式传输的组合在 Agent 系统中是成熟方案。
但在工程化层面存在若干可改进的点。

## 二、核心问题（按优先级排序）

### P0 — 工具注册链路曾经过于分散（现已统一）

当前本地工具已统一收口到：

```
@tool() → runtime/discovery.py → tools.bootstrap → ToolRegistry / runtime.dispatcher / permissions → docs/tools.md
```

一致性校验已由 `check_tool_consistency()` 覆盖，历史兼容注册链路已移除。

### P0 — 工具错误处理不统一

部分工具返回 `ToolExecutionResult`，部分返回裸字典 `{"error": "..."}`，部分直接 raise。
Agent 层需要兼容多种格式，增加了脆弱性。

**升级方向**：在 dispatcher 层包装统一的错误处理中间件，确保所有工具输出都是
`ToolExecutionResult`，异常自动捕获转换。

### P1 — 工具执行缺乏超时保护

除 `execute_code` 有 30s 超时外，其他工具没有超时机制。
一个卡死的工具会阻塞整个 ReAct 循环。

**升级方向**：在 dispatcher 层加统一的超时包装，默认 30s，可按工具配置。

### P1 — 上下文压缩失败无降级

当 LLM 摘要调用失败时，直接抛 `ContextCompressionError` 终止任务。
对于长对话场景，这是一个致命路径。

**升级方向**：增加降级策略 — 摘要失败时回退到简单截断（保留最近 N 轮，丢弃最早的消息），
保证任务不中断。

### P1 — SSE 事件队列无背压/丢失风险

`SSEAdapter` 使用 `put_nowait()` 放入队列，队列满时事件直接丢失，仅记录 warning。

**升级方向**：
- 关键事件（生命周期类）使用有界队列 + 阻塞写入
- 流式增量事件允许丢弃但记录 gap 序号，前端可检测缺失

### P2 — 占位符路径解析脆弱

`{result_1.content.layers}` 路径不存在时返回原占位符字符串，后续 JSON 解析会失败。

**升级方向**：路径解析失败时返回明确的错误标记
（如 `{"__ref_error__": "path not found"}`），让 Agent 能感知并重试。

### P2 — 观察文本截断过于粗暴

子 Agent 结果摘要硬截断 500 字符，可能截断关键数据结构（如 JSON 中间断开）。

**升级方向**：改为结构感知截断 — 优先保留 `summary` 和 `answer` 字段，
`content` 按需截断，保证 JSON 完整性。

### P3 — Artifact 无版本历史

`revise_visualization` 直接覆盖原文件，版本号简单递增，无法回滚。

## 三、数据传输层评估

| 层级 | 现状 | 评分 | 说明 |
|------|------|------|------|
| LLM → XML 解析 | 增量状态机 + 多层容错 | 9/10 | 裸占位符修复、反斜杠路径修复实用 |
| 工具执行 → 结果 | ToolExecutionResult 模型 | 7/10 | 模型设计好，但使用不统一 |
| 结果 → 占位符 | 两层替换（Agent层+工具层） | 8/10 | 设计精巧，路径解析需加固 |
| EventBus → SSE | 会话隔离 + 心跳保活 | 8/10 | 缺背压机制 |
| SSE → 前端 | 事件分发 + 重连回放 | 8/10 | 重连机制完善 |

## 四、升级路线

### Phase 1（低风险高收益）

在现有代码上加包装层，不需要重构，风险最低。

1. **统一工具错误处理中间件** — dispatcher 层包装
2. **工具执行超时保护** — dispatcher 层包装
3. **上下文压缩降级策略** — pipeline 层增加 fallback

### Phase 2（中等工作量）

需要引入新的注册机制，涉及多文件改动。

4. **统一工具注册框架** — 装饰器 + 自动发现
5. **SSE 关键事件背压保护** — SSEAdapter 改造
6. **占位符路径解析加固** — result_references 改造

### Phase 3（长期优化）

7. **大结果分离存储** — 超过阈值存临时文件，传引用
8. **观察文本结构感知截断** — 结果摘要改造
9. **Artifact 版本历史** — 版本管理器
